# second-brain

A living knowledge base built alongside an AI, tracking my journey from software engineer to AI engineer.

## What this is

This repo is not a course. It is not a set of notes taken from a tutorial. It is a knowledge base that grows in real time as I learn — concepts added when understood, updated when understanding deepens, cross-linked when connections emerge.

Every file here was either written by me, written by an AI that was teaching me, or written collaboratively. The goal is that at any point I can drop this repo into a Claude Project or feed it to any AI and it instantly knows where I am, what I understand, and what I should learn next.

## Folder structure

```
foundations/        core concepts — tokens, embeddings, context, prompting
rag/                retrieval-augmented generation and vector search
pipelines/          LangChain, agents, tool use, evals
model-internals/    transformers, attention, fine-tuning
production/         deployment, latency, cost, observability, safety
projects/
  second-brain/     the project itself — architecture, decisions, code notes
meta/               learning status, roadmap, what to learn next
```

## How to use this with an AI

Drop this repo into a Claude Project or paste `BRAIN.md` as a system prompt. The AI will immediately understand your background, your current phase, and your knowledge gaps — no re-explaining needed.
